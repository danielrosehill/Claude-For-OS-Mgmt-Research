Contrast tool. Open Interpreter explicitly targets "let an LLM run code on your own machine" as
its primary framing; the user is fully responsible for review, there is no project-scoped
sandbox, and admin-shaped tasks (data wrangling, file manipulation, system administration) are
first-class in the positioning. Useful for the research as an existence proof of the alternative
stance — an ops-oriented default — against which to diff Claude Code's developer-tool default.
Not a recommendation, just a framing contrast for the deep-dive on "what a first-class authorised
operator mode would look like."
