Second practitioner post framing Claude Code as "a diagnostic partner, not an autonomous
executor." Advocates hypothesis/test/observe loops rather than letting the agent freely execute,
explicitly because of hallucination risk and lack of organisational context. Doesn't dig into the
permission model itself, but is a useful datapoint for the "what 'good' sysadmin use of Claude
Code looks like" framing — and for the counter-argument that heavy oversight is still needed even
when sandbox friction is removed.
